# 赛题对齐与差距分析

## 验收基线

赛题重点不是一次性生成建议，而是形成“问题理解 - 知识整合 - 假设生成 - 证据梳理 - 研究计划 - 反馈修正”的闭环。初赛重点检查：

1. 闭环链条完整。
2. 假设有真实文献或数据证据。
3. 研究计划可执行、可证伪。
4. 数据或人工反馈真实影响下一轮。
5. 迭代过程清晰且质量可比较。
6. 提供可调用测试 API、交互前端、案例、技术报告和源码。

## 初始项目不足

| 领域 | 初始状态 | 风险 |
| --- | --- | --- |
| 总控 | 只有单阶段 Agent 网关 | 没有任务状态机，无法证明闭环 |
| 上下文 | 主要保存在 React 状态 | 刷新丢失，无法审计 |
| 审核 | 前端生成模拟审核分数 | 评测 API 无法复现审核逻辑 |
| 反馈 | 前端重跑但不形成服务器版本 | 无法证明反馈改变下一轮 |
| Artifact | 只有设计文档和静态列表 | 阶段输入输出没有统一落盘 |
| MCP | 未实现 | 无法安全地向外部 Agent 暴露产物 |
| 统一规范 | 仅做字段适配 | 缺少机器可读注册表、写入边界和统一追踪 |
| API | 只有 health 和无状态 stage run | 不满足任务、版本、事件、导出要求 |

## 本轮完成

- 新增任务级 Orchestrator、状态流转和三种运行模式。
- 新增 Pydantic 统一响应契约和 JSON Schema。
- 新增 Agent 注册表、上下文切片和字段写入白名单。
- 新增 Review Gate，检查格式、必填字段、下游可用性和证据 ID 追溯。
- 新增受限 Artifact Service、事件日志、版本快照、字段级 diff 和 ZIP 导出。
- 新增官方 SDK MCP 服务，且不开放任意文件系统路径。
- 前端改用任务级 API，并增加真实运行/Artifact/API/版本差异面板。
- 增加总控、路径安全、反馈迭代和版本差异自动化测试。

## 仍需后续完成

| 优先级 | 工作 | 验收方式 |
| --- | --- | --- |
| P0 | 准备至少一个来源完全真实的代表性案例 | 文献 DOI/URL 可访问，输出无占位引用 |
| P0 | 执行一次反馈后的第二轮真实 Agent 调用 | V1/V2 diff 显示假设或计划实质变化 |
| P1 | 接入公开数据分析或虚拟实验 Runner | 数据结果写入 feedback event 并改变计划 |
| P1 | 自动生成技术报告 Markdown/PDF | 导出包包含报告、案例和 API 说明 |
| P1 | 增加生产鉴权和数据库 | 非本机部署时隔离不同用户任务 |
| P2 | 增加假设树搜索和锦标赛操作 | 支持 expand/refine/discard/merge/rerank |

这份清单应作为提交前检查表，不应把 mock 文献或未执行实验描述为真实结果。
