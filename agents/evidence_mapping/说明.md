
## 交付内容

路径：`evidence_mapping_agent/`

这是模块 4「证据梳理 Agent」v0.1 可运行实现，已对齐 `数据规范_v0.1`：

- 统一输出外壳：`metadata` + `payload.evidence_map` + `self_review`
- 支持 / 反对 / 不确定三分类（互斥）
- 独立重判方向（`recheck_delta` 记录相对模块 2 的改判）
- 四维评分 + 缺口 + `rollback_target`（给总控回退用）
- JSON Schema、`mock_input`、一键 Demo、单元测试

## 如何验收（1 分钟）

```bash
cd evidence_mapping_agent
pip install -r requirements.txt
python examples/run_demo.py
python -m pytest tests -q
```

预期：测试通过；`hyp_001` 能看到 support/oppose；`ev_005` 相对模块 2 hint 被改判为 oppose。

## 总控怎么接

```python
from evidence_mapping import EvidenceMappingAgent
# 或参考 examples/adapter_example.py
resp = EvidenceMappingAgent().run_dict(input_slice)
task_context["evidence_map"] = resp["payload"]["evidence_map"]
```

请按 `verdict.rollback_target` 映射：

- `knowledge_integration` → 回退模块 2
- `hypothesis_generation` → 回退模块 3
- `none` → 进入模块 5

## 需要会上确认的上游字段

1. 模块 2：`evidence_id / claim / quotes / literature_id`
2. 模块 3：`expected_observation` 或 `predictions[]`
3. 前端：Evidence Map 三列 + 强度分 + 缺口
